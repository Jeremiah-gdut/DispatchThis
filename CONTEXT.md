# DispatchThis

DispatchThis 是以实战恢复为首要目标、从已知 ARM64 ELF 混淆二进制中恢复可读控制流的
Binary Ninja 工作流插件。公开教学和第三方扩展只在不削弱真实样本恢复时兼顾。

## 术语

**实战恢复**：
以用户正在分析的已知二进制获得可信、可核验的恢复结果为首要成功标准。
_避免使用_：教育性 POC、通用反混淆平台

**样本**：
由一个独立样本插件支持的单个 ARM64 ELF 混淆二进制；不承诺同家族或相似二进制可复用。
_避免使用_：样本家族、泛化目标

**DispatchThis 核心**：
负责工作流阶段、顺序及时序编排，并提供与样本无关的功能型 API 的 Binary Ninja 外部插件。
它不拥有任何具体样本的识别规则、目标公式、加密算法或平坦化结构。
_避免使用_：通用反混淆器、样本实现

**样本插件**：
仅支持一个具体样本、调用 DispatchThis 核心能力并注册该样本语义提供者的独立外部插件；
它不创建或插入 workflow activity，新增样本意味着编写新的样本插件。
_避免使用_：解析 profile、样本家族插件、bundled profile

**样本语义**：
只对一个样本成立的识别与恢复知识，例如跳转目标计算、字符串加解密算法和控制流平坦化
结构。

**样本语义提供者**：
由样本插件向 DispatchThis 核心注册的纯语义实现。它返回恢复事实或计划，不直接修改
Binary Ninja、工作流阶段状态或回执；它只能实现或省略核心定义的固定阶段槽位。
_避免使用_：workflow callback、mutation backend

**SampleSemantics**：
由 DispatchThis 核心定义、样本插件传给 `register_provider(...)` 的样本语义描述对象。它集中
携带稳定的 provider 身份、写死的核心 API 版本和六个具名可选 callable；未提供某个 callable
表示该样本不支持对应槽位。每个 callable 只接收对应的核心槽位 Query。它不能携带自定义阶段、
顺序、门控或 Binary Ninja 修改逻辑。
_避免使用_：通用 `provide(request)` 分派器、分散的 callback 注册

**核心 API 版本**：
外部样本插件在 `SampleSemantics.api_version` 中以整数字面量声明其编写时所针对的 DispatchThis
公开契约版本；核心只注册精确匹配的提供者。公开 Query、结果类型或功能型 API 的破坏性变更
递增该版本，内部实现或兼容性增加不递增，也不提供版本范围、兼容适配器或旧接口 shim。
_避免使用_：插件发布版本、自动导入当前版本、语义化版本范围、兼容层

**槽位 Query**：
DispatchThis 核心为六个样本语义槽位分别定义的冻结输入对象。它只携带该槽位必需的原生
`BinaryView`、`Function`、当前 LLIL/MLIL 和核心提供的只读证据；其中原生 Binary Ninja
对象仍具备修改方法，但样本插件必须按只读契约使用。Query 绝不暴露 `AnalysisContext`、
Settings、workflow state、receipt 或任何 session data。
_避免使用_：可变参数字典、AnalysisContext 透传、工作流状态快照

**槽位结果类型**：
由 DispatchThis 核心定义并拥有的六种冻结强类型数据对象：`BranchTargetFact`、
`CallTargetFact`、`GlobalDataFact`、`CorrelatedStorePlan`、`StringRecoveryFact` 和
`DeflattenPlan`。它们只携带恢复证据、当前 IL 见证或声明式修改意图，不携带执行修改的
callable。样本插件不得用裸字典、任意元组、自定义类或子类替代这些类型。
_避免使用_：plan dict、插件私有 result、mutation callback

**槽位调用结果**：
每个已实现的样本语义 callable 必须返回 `CompleteBatch[T]` 或 `Inconclusive(reason)`。
`CompleteBatch(())` 表示本轮规则完整执行但没有结果；非空完整批次才可进入核心验证与应用。
`Inconclusive` 表示本轮无法完成证明，核心不接受任何部分结果、不关闭 receipt、不标记阶段
稳定，并阻止依赖该阶段的下游 pass。间接分支目标的批次完整性描述完整扫描及每个已返回站点
事实的完整性，不要求所有候选站点均已证明；一个已完整证明、但当前后端不能应用的多目标
调用仍是 `CompleteBatch` 中的事实，而不是 `Inconclusive`，也不阻止调用阶段收敛。callable
缺失单独表示 provider 不支持该槽位。调用目标批次还允许省略当前 provider 不支持的调用；
只有整轮扫描本身不可信时才返回 `Inconclusive`。
_避免使用_：用空列表同时表示无结果、失败和不支持

**分支目标逐站点进展**：
间接分支目标槽位以 `CompleteBatch[BranchTargetFact]` 返回一次完整扫描中已完整证明的站点事实；
没有 fact 的候选由核心根据当前 unresolved metadata、CFG 和已有 receipt 推导为未决，不由
provider 重复列举。核心立即应用已证明事实，但只要仍有未决站点就保持 branch phase 未稳定；
每个事实必须包含该站点的完整目标集合。
_避免使用_：独立 `BranchTargetBatch`、重复 unresolved 列表、best-effort 目标子集

**提供者注册入口**：
样本插件与 DispatchThis 核心之间唯一的注册 seam：`register_provider(SampleSemantics(...))`。
核心先精确验证 `api_version`，不匹配时拒绝整个提供者并输出一次明确错误；匹配后才从描述
对象取得具名槽位。它不接受样本插件逐项修改全局 callback 或 workflow。
_避免使用_：每槽位注册函数、workflow activity 注册

**工作流所有权**：
DispatchThis 核心创建、排序并注册 workflow 和 activity，以及执行阶段门控和失效传播的
独占职责；样本插件不拥有其中任何部分。

**固定阶段槽位**：
由 DispatchThis 核心定义名称、输入、顺序、依赖和失效规则的恢复生命周期位置。样本语义
提供者可以实现或省略槽位，但不能新增、插入或重排它们。
_避免使用_：样本自定义阶段、动态 phase DAG

**样本语义槽位**：
样本语义提供者可选实现的六个 interface：间接分支目标、间接调用目标、全局数据语义、路径
关联 STORE 恢复、字符串恢复和去平坦化重定向。每个槽位只返回核心定义的恢复事实或计划。
_避免使用_：自定义 pass、任意 workflow hook

**路径关联 STORE 恢复**：
当两个控制流前驱分别为 STORE 的目标和来源建立成对 PHI 输入、而汇合后的 MLIL 掩盖这层
路径对应关系时，将经完整证明的具体 STORE 移回各自前驱，并原子 NOP 汇合 STORE 的可选
恢复 pass。样本插件识别样本结构并返回计划；核心重新验证当前 IL 见证并执行 copy-transform。
_避免使用_：通用 STORE 优化、独立组合 PHI 候选、跨函数内存恢复

**核心恢复实现**：
核心在样本语义槽位之外拥有的恢复行为，包括分支条件翻译、目标解码清理、类型调整、注释
写入、原子 MLIL copy-transform、回执、失效传播和重新分析修改。
_避免使用_：样本算法、样本语义槽位

**运行时提供者绑定**：
核心 activity callback 执行时才取得当前样本语义提供者，使外部插件加载顺序不会要求修改
已注册且不可变的 workflow。
_避免使用_：样本 activity 注入、注册前静态绑定

**显式提供者选择**：
多个外部样本插件可以同时注册提供者；DispatchThis 核心从自身拥有的设置中读取稳定
provider ID，并在运行时绑定对应 `SampleSemantics`。核心不自动识别样本，也不因当前只有
一个已注册提供者而隐式选择。选择入口是核心注册的唯一 BinaryView 插件菜单
`DispatchThis\\Select Provider…`；菜单被调用时才动态列出当前 registry，不为每个 provider
注册独立命令。provider 选择不进入 Function Settings，也不由样本插件提供独立菜单或状态。
_避免使用_：自动探测、唯一提供者回退、样本插件私有选择状态

**BinaryView 提供者绑定**：
核心插件菜单取得当前 `BinaryView`，将选中的 provider ID 以该视图为 resource 写入
`SettingsResourceScope`，使其随 Raw BinaryView/BNDB 持久化。核心 workflow callback 使用
`AnalysisContext.view` 读取该 ID，并在运行时从核心 registry 取得对应 `SampleSemantics`。
provider 身份不存入函数设置、`Function.session_data` 或 `BinaryView.session_data`。
_避免使用_：函数级 provider override、仅存于内存的活动 provider

**功能型 API**：
供样本插件复用的、与样本无关的 BNIL、数据流、变量定义和 PHI 分析能力。它应覆盖反复出现
的边界情况，同时允许样本插件组合较低层能力表达特殊语义。公开 surface 分为基础原语与
完整证明两层，但不包含样本规则 DSL、通用模式引擎或具体样本识别器。
_避免使用_：profile helper、样本规则 API

**基础原语 API**：
功能型 API 的低层逃生口，提供变量身份、SSA 定义、PHI、表达式遍历、内存读取、CFG、
支配关系和当前 IL 见证等精确操作。它直接接受原生 Binary Ninja 对象，不包装成另一套 IL。
_避免使用_：样本 helper、Binary Ninja 对象模型替代层

**完整证明 API**：
建立在基础原语之上的深层通用分析，包括递归定义图、完整多值求解、调用目标切片、别名与
地址逃逸、路径私有性和状态写入证明。它要么返回完整证据，要么显式未知，绝不静默截断或
返回已知子集。
_避免使用_：best-effort solver、样本识别器、规则 DSL

**完整定义图**：
从一个 BNIL 表达式或变量出发，按真实变量和指令身份递归收集全部到达定义、每个 PHI 输入、
语义叶节点、边和循环的核心证明结构。遍历不设置隐藏的固定深度上限；任一必要定义缺失或
无法读取时，依赖完整性的消费者得到显式未知。
_避免使用_：第一个定义、固定 max_depth、按显示名称去重

**路径来源**：
在 Binary Ninja 当前 CFG 中，把一组 PHI 取值归属于同一控制流经过关系的来源。一个来源
可以包含多个汇合点的相容入边选择；它不判断分支谓词在运行时是否可满足。
_避免使用_：PHI 操作数位置、前驱块编号、值相等关系、路径可行性证明

**路径关联值关系**：
在当前 CFG 给出的路径范围内，以路径来源约束多条 PHI 的完整联合取值关系。重复的完整
取值可以合并，但必须保留其全部路径来源；它不识别或排除虚假分支。
_避免使用_：独立 PHI 值集合的笛卡尔积、提前去重的 PHI 候选、不透明谓词求解

**路径关联 PHI 求值**：
间接跳转目标恢复在当前 IL 中使用路径关联值关系计算完整目标集合的求值过程。内部 PHI case
和 CFG 入边关系只服务于本次目标计算，不跨重新分析持久化；但当 deinbr 据此证明的是条件
跳转时，其最终恢复事实必须保留混淆前的语义条件以及有方向的 true/false 目标，供后续分支
条件翻译直接消费。
_避免使用_：持久路径证据、跨阶段 PHI case、只保留无方向目标集合

**条件分支目标事实**：
deinbr 对一个已证明具有两个不同有向目标的间接跳转给出的核心拥有结果，表达跳转源、可在该
恢复点直接使用的语义条件、`true_target` 与 `false_target`。`targets` 是提交给 Binary Ninja
的无方向去重投影；分支条件翻译只消费事实，不重新求目标、发现条件或猜测方向。
_避免使用_：无序二目标事实、selector 等值比较代替原始语义条件、translator 内目标解码

**无需条件翻译的分支事实**：
`BranchTargetFact.condition is None` 的目标恢复事实，明确表示该站点不应还原为 if/else；它
仍可提交完整目标，但不创建条件回执、不运行 translator，也不计入条件翻译完备门。目标数量
本身不能决定此语义，即使恰有两个不同目标也不自动视为条件分支；相同的 true/false 目标则
必须投影为这种事实。
_避免使用_：条件恢复失败、按双目标自动推断条件、静默降级

**退化条件目标**：
路径关联求值为条件的 true/false 两侧证明出同一具体目的地的结果。provider 保留本次求值中的
全部路径来源，但只返回去重后的完整目标及 `condition=None`；非空 condition 与相同有向目标
的组合不是合法的 `BranchTargetFact`，也不会创建条件回执。
_避免使用_：同目标 IF、退化条件回执、不完整目标

**恢复点条件**：
deinbr 随条件分支目标事实交付、可在对应间接跳转站点直接构建 `MLIL_IF` 的原语义条件见证。
样本语义提供者保证其在恢复点的语义有效性；核心只重新绑定并验证当前形态，不重复证明副作用、
支配或路径安全。无法保证时该站点保持未决，translator 不搜索上游 IF 或合成条件。
_避免使用_：上游路径模式、selector 等值条件、translator 条件发现

**Condition 根见证**：
恢复点条件在当前 LLIL 中的单一根表达式见证；根下可以包含任意嵌套子表达式，也可以读取定义
位于其他指令的变量。核心不把它拆成表达式列表或展开定义链，只将该根唯一映射到当前 MLIL。
_避免使用_：单叶条件、表达式列表、内联定义图、条件 AST

**分支条件翻译**：
核心消费条件分支目标事实，在同一跳转站点将 Binary Ninja 已恢复目标的 switch-like
`MLIL_JUMP_TO` 原位替换为表达恢复点条件的 `MLIL_IF`。它不负责目标求解、条件发现、上游 CFG
分析或路径出口改写，也不把 HLIL 展示形态作为翻译证据。
_避免使用_：间接跳转解析、switch selector 条件、通用控制流简化

**逐站点条件翻译**：
每个条件分支目标事实分别重新绑定和验证。可成功翻译的站点进入本轮原子 MLIL copy-transform；
单个站点绑定或验证失败只保留该处原 switch-like 分支并产生可见诊断，不能取消其他站点的成功
翻译。只有实际安装成功的站点才能提供后续目标解码清理证据；失败站点的解码 IL 必须保留。
_避免使用_：函数级全有或全无、静默跳过失败站点、清理未翻译站点

**条件翻译失败诊断**：
核心只为已有条件回执、且能归责到单个条件站点的站点绑定、条件绑定、目标验证或当前形态失败，
在其源地址维护自动标签，并只在失败首次出现或原因变化时输出 warning。站点随后翻译成功或
其条件回执失效时移除标签；重复出现的同一失败不重复刷日志。
_避免使用_：静默失败、用户标签、每轮分析重复 warning

**ConditionTranslationFailure**：
核心内部的冻结站点失败结果，由源地址、`ConditionFailureReason` 枚举和只供显示的 detail
组成。原因枚举参与比较与去重，detail 不参与；会话状态只保存枚举的稳定值，不保存可能因
热重载失效的枚举对象。
_避免使用_：裸原因字符串、见证角色层次、以日志文本去重

**已满足条件翻译**：
当前 IF 直接使用条件回执重新绑定并唯一映射得到的精确条件见证，同时精确表达 true/false
目标方向，因而无需再次改写的站点状态。逻辑等价但不是该见证的表达式不满足此状态；后续
清理仍须从当前 MLIL 重新证明。
_避免使用_：仅目标集合相同、逻辑等价条件、重复改写、复用旧清理索引

**当前条件翻译状态**：
translator 针对本轮 `ctx.mlil` 重新计算的状态：同源 switch-like `MLIL_JUMP_TO` 的完整目标
投影与回执一致时为 `REWRITE_READY`；同源 `MLIL_IF` 精确使用重绑条件及有方向目标时为
`ALREADY_SATISFIED`；其他形态、目标不一致或映射缺失/歧义均为 `FAILED`。该状态只支配当前
overlay 的翻译、清理与 deflatten 门控，不跨重新分析保存，也不通过 HLIL 或其他 CFG 形态
回退恢复。
_避免使用_：`translation_done`、跨 MLIL generation 的成功缓存、旧 overlay 状态

**条件翻译批次失败**：
多个已验证站点共同执行的原子 MLIL copy-transform 或安装失败。它是函数级恢复后端错误：核心
输出一次 error 并保持条件翻译完备门关闭，但不把同一后端故障标记到每个源地址。
_避免使用_：站点绑定失败、批量地址标签、部分安装

**条件翻译完备门**：
逐站点成功翻译不会因其他站点失败而撤销，但 deflatten 只有在当前函数的全部有效条件回执均已
翻译、且分支清理达到当前固定点后才可运行。`condition=None` 的分支事实不进入该集合；任一
失败条件站点保持函数级门关闭。
_避免使用_：按区域 deflatten、失败站点回滚成功翻译、计划到站点依赖图

**条件事实交接**：
核心在间接跳转解析与分支条件翻译之间保存条件分支目标事实的稳定语义身份，并在重新分析后
重新绑定当前跳转站点和恢复点条件。deinbr 未形成条件回执时仍属于间接分支目标 phase；
若原因是合法的 `condition=None`，translator 直接忽略；若该站点未决，branch phase 保持
未稳定，但其他已证明站点仍可取得进展。translator 不为从未收到的事实产生失败诊断。样本
插件只证明样本语义，不拥有跨阶段状态或重新分析时序。
_避免使用_：样本插件回执、旧 IL 对象、translator 重新求值

**条件回执**：
条件事实交接在当前函数会话中的核心状态，以稳定标量保存跳转源、恢复点条件定位以及有方向
目标。它不构成可跨 BNDB 重开的持久证明；新会话必须由 deinbr 取得新鲜事实。
_避免使用_：持久 IL 定位缓存、旧会话证明、从无方向目标重建条件

**条件回执失效**：
仅当 provider 绑定、相关 pass 开关、所属分支目标回执或同源新鲜条件事实真实改变，或者函数
会话结束时移除条件回执。增量扫描省略已验证 source、当前 IL 重绑失败、形态不支持、原子安装
失败或站点已翻译为 IF 都不构成失效；这些状态仍由原回执支配诊断和 deflatten 门控。
_避免使用_：失败即删除回执、扫描省略即失效、成功即遗忘

**ILAnchor**：
核心从当前跳转或条件见证提取的窄定位信息，以机器来源、相对操作数路径、预期 operation 和
位宽在重新分析后唯一绑定当前见证。它只服务于站点与恢复点条件的重绑，不表示或复制表达式
语义，也不是通用 IL 序列化层。
_避免使用_：旧 IL 索引、字符串化 IL、序列化表达式树、样本插件定位器

**精确条件见证**：
condition 根 `ILAnchor` 在当前 LLIL 中唯一重绑、再唯一映射得到的当前 MLIL 根表达式。
translator 深拷贝该根及其嵌套子表达式，但不复制或内联其变量定义；它只按当前表达式身份安装
或识别 IF，不接受由布尔化简、代数改写或文本比较判定的等价式。
_避免使用_：条件等价求解器、表达式指纹、`str`/`repr` 比较

**分析预算**：
调用者显式提供的确定性节点/边上限，只用于约束完整定义图和其他深层证明的资源消耗，不是
语义深度。预算耗尽返回 `Inconclusive`，绝不把已访问部分作为完整结果；循环由身份去重显式
记录，不依赖预算截断。
_避免使用_：隐藏递归上限、超时后返回部分值

**ValuePolicy**：
样本插件向核心完整多值求解器提供的纯求值策略，只处理核心不认识的样本特有运算或受控
内存读取。核心拥有定义图、PHI、位宽、符号性、标准 BNIL 运算、循环和完整性；policy 只
接收当前表达式及已经完整求出的操作数值，并返回 `Handled(values)`、`NotHandled` 或
`Inconclusive(reason)`。它不能自行递归定义、修改 Binary Ninja，或存入恢复事实和计划。
_避免使用_：样本私有值引擎、递归 evaluator、mutation policy

**解码 gadget**：
从编码数据在运行时计算控制流或调用目标的一小段混淆序列。

**间接跳转解析**：
恢复计算跳转的具体目标，使分析可发现下一个基本块。
_避免使用_：deinbr

**MLIL 功能型 API**：
面向 Medium Level IL 的功能型 API，例如调用遍历、表达式标量值提取、表达式操作查询和
考虑变量定义的表达式遍历。它不包含样本识别器、模式规则或恢复算法。
_避免使用_：字符串解密 helper、解析引擎

**表达式标量值**：
从一个 MLIL 表达式直接恢复的常量或 Binary Ninja 单值结果，不进行算术折叠、内存读取、
PHI 候选展开或目标校验。
_避免使用_：完整值引擎、常量折叠器

**表达式操作查询**：
helper 级检查，用于判断 MLIL 表达式树（可选地包括跟随的变量定义）是否含调用者提供的
原生 MLIL operation enum 或 enum 派生兼容名称之一。
_避免使用_：模式匹配器、字符串解密识别器

**函数工作流启用状态**：
由核心插件菜单为当前函数切换、以该 `Function` 为 resource 持久化的 pass opt-in 设置；它
决定对应核心工作流阶段是否运行，但不定义样本语义。Function Analysis Settings 是底层状态
与 workflow eligibility 机制，不要求用户通过设置面板操作。
_避免使用_：整视图工作流应用

**逐 pass 开关**：
间接分支目标恢复、间接调用目标恢复、全局数据语义恢复、分支条件翻译、路径关联 STORE
恢复、字符串恢复和去平坦化分别拥有 `register_for_function` 插件菜单开关；每个菜单项写入
对应函数级 ResourceScope 设置并安排该函数重新分析。目标 cleanup、回执、失效传播、类型
调整、注释写入和原子改写是所属 pass 的正确性后端，不构成可单独关闭的 pass。
_避免使用_：合并 resolver 开关、内部回执开关

**Pass 依赖闭包**：
函数的已启用 pass 集合必须包含每个已启用下游 pass 的全部传递前置依赖。菜单开启下游时
同步开启缺失的前置 pass；关闭上游时同步关闭其全部依赖者并使对应下游证据失效。核心绝不
执行菜单状态显示为关闭的 pass。最小直接依赖为：call target 依赖 branch target；global data
依赖 call target；branch translation 依赖 global data；correlated STORE 与 string 分别依赖
global data；deflatten 依赖 branch translation。deflatten 不依赖 STORE/string，STORE、string
与 translation 也不互相依赖；它们仅在同时启用时服从固定 activity 顺序。cleanup、receipt 和
稳定性条件是内部 readiness gate，不是开关依赖边。
_避免使用_：隐式运行已关闭 pass、永久阻塞的无效开关组合

**恢复事实**：
样本语义提供者返回给 DispatchThis 核心的一项标准恢复分析信息，如间接分支目标、间接调用
目标或全局数据类型。核心工作流决定何时及如何将其提交给 Binary Ninja。
_避免使用_：样本插件修改请求、Binary Ninja mutation request

**恢复目标集**：
当前恢复证据支持的完整具体控制流目的地集合。消费者必须保留每个目标，除非证据证明
恰好只有一个目标有效。
_避免使用_：第一个目标、首选目标

**隐式目标剪枝**：
未作语义证明便静默选择恢复目标集中的一个成员。它会抹去有效 CFG 边，因此绝不是安全的
单目标便利操作。
_避免使用_：pick first、best-effort target

**已验证分支前沿**：
其完整回执目标元组与 Binary Ninja 当前非自动用户分支元数据完全一致的分支源。只有这些
源可从增量识别中省略；仅有回执、自动、缺失、子集、超集或已变更映射的源仍在解码前沿。
_避免使用_：缓存分支、假定已解析源

**完整值证据**：
一个 BNIL 表达式每条语义路径支持的全部具体值。helper 要么返回完整集合，要么显式未知；
不支持的臂、受限展开遗漏、无法读取的 load 或无效目标均不能许可已知子集结果。
_避免使用_：best-effort values、仅已知分支

**当前 IL 见证**：
恢复计划保留的 Binary Ninja 指令对象，在修改边界通过精确 operation、地址、
instruction/expression 索引、相关操作数和所属 IL 函数重新绑定。扫描到的相似指令并非
同一个见证。
_避免使用_：附近匹配、过期计划对象

**混合 IL operation-name 接缝**：
匹配器有意同时处理 LLIL 与 MLIL 时使用的窄兼容边界，借由 enum 派生名称避免相等的
`IntEnum` 值跨层混用。只处理单一 IL 的模块使用 Binary Ninja operation enum。
_避免使用_：手写 MLIL_* 字符串、手写 LLIL_* 字符串

**全局数据恢复事实**：
`GlobalDataFact` 标识精确全局数据槽位，并携带样本 provider 已完整证明的原生 Binary Ninja
`Type`。该类型可以表达指针、数组、结构体及各层 const；核心只验证当前地址、映射范围、类型
宽度、事实冲突和应用后的精确读回，不接收类型字符串，也不猜测或改造 provider 的类型。
_避免使用_：全局常量审计记录、类型名字符串、核心推断类型、全局数据修改回执

**恢复后端**：
消费恢复事实并应用稳定 Binary Ninja 分析效果（CFG 恢复、调用目标应用、IL 翻译、全局
槽位类型或清理）的核心层。样本语义提供者为其产生输入；功能型 API 不拥有这些修改。
_避免使用_：样本语义提供者、功能型 API

**间接调用解析**：
恢复一个间接调用可能到达的完整具体 callee 集合，并在当前后端支持时将其应用到调用点。
_避免使用_：deincall

**调用目标事实**：
`CallTargetFact` 对一个当前间接调用给出的完整恢复证据，以非空、去重的 `targets` 元组保存
所有具体 callee。事实是否完整与当前恢复后端能否应用该基数是两个独立判断。
_避免使用_：单一首选 callee、裸 target 整数、部分候选

**保留的多目标调用**：
`CallTargetFact.targets` 含多个 callee、因而首版核心不能把 `call.dest` 安全替换为单一常量的
站点状态。核心保留原间接调用及其活目标解码，不创建调用回执或 cleanup 义务，也不阻止调用
阶段收敛；下游证明只有在真实依赖该调用时才局部拒绝。核心不把这种事实降格为
`Inconclusive`，也不经变量定义级 UIDF 猜测站点局部语义。
_避免使用_：调用阶段阻塞项、pick-first call、伪 direct call、部分调用回执

**调用阶段收敛**：
provider 本轮扫描可信、全部受支持的单目标调用改写与类型调整均已读回，且当前没有新的调用
修改会触发重新分析的状态。它不表示函数中所有间接调用均已变为 direct call，也不包含独立的
调用目标 cleanup 固定点；保留的多目标或其他未支持调用不妨碍该状态。
_避免使用_：全部调用已解析、无间接调用、调用 cleanup 完成

**调用目标扫描完成**：
调用 provider 已检查本轮全部当前候选并返回 `CompleteBatch[CallTargetFact]` 的状态。每个返回
事实都必须包含所属站点的完整 callee 集合，但批次可以省略不匹配或当前 provider 未支持的
调用；这些调用保留原 IL 且不阻塞。预算耗尽、遍历失败或其他整轮可信性缺失必须返回
`Inconclusive`，不能伪装成扫描完成。
_避免使用_：所有调用均已恢复、省略即未决、失败时返回空批次

**调用目标定义切片**：
馈入 `call.dest` 的完整当前 SSA 到达定义链，包含每个 PHI 输入。只有映射到当前非 SSA
赋值的精确整变量 SSA 定义属于切片。替换目标后，只有该自有切片中其 SSA 值无其他消费者
的赋值可清理；xref 不定义所有权。
_避免使用_：前序指令、整函数 dead-load 扫描

**全局数据语义恢复**：
将样本 provider 已证明的完整全局数据类型交给核心验证并应用的固定阶段。样本插件拥有具体
数据语义与类型证明；核心独占 `define_user_data_var`、回执、失效和重新分析时序。
_避免使用_：全局常量解析、全局变量修复、任意类型字符串应用

**字符串解密**：
从样本的编码字节 blob 中恢复明文字符串。
_避免使用_：通用字符串反混淆

**字符串解密函数**：
样本的 decoder clone：向调用者提供的 buffer 写入一个明文字符串，并设置一次性完成
标志。
_避免使用_：字符串 helper、通用 decoder

**解密字符串注释**：
包含字符串解密函数调用恢复出的明文的 Binary Ninja 调用点注释。
_避免使用_：恢复字符串字面量

**字符串解密恢复事实**：
一个字符串解密调用点的标准恢复信息：调用地址、源 blob 地址、目标 buffer 地址和明文字节。
工作流代码负责将其变为解密字符串注释。
_避免使用_：注释计划、profile annotation

**调度器**：
由状态值选择下一个原始基本块的平坦化控制流路由器。

**调度器簇**：
一组以状态令牌比较进行路由的连通调度器比较块。通过变量/常量相等、不等或有符号/无符号
有序比较识别，其变量必须有唯一的行内直接复制链并结束于同一状态输入；图形结构验证该簇，
但不是主要信号。

**调度器比较链**：
同一调度器行中更早出现的、馈入其比较并结束于共享状态输入的唯一直接等宽变量复制序列。
其他位置追溯到状态的任意定义并不等价，因为该比较值可能在另一入口路径上过期。

**精确整变量读取**：
读取完整变量的 `MLIL_VAR` 或 `MLIL_VAR_SSA` 使用。字段、split 和 aliased 形式仍是重要的
may-read/may-alias 证据，但不能证明调度器比较或指针复制传递完整值。
_避免使用_：将 `VAR_FIELD` 视作直接复制

**变量身份**：
显式规范化 wrapper 后的 Binary Ninja 底层 Variable、SSAVariable 或 register 对象的
equality/identity。显示名不是身份，因为不同存储对象可有相同 `str`/`repr`。
_避免使用_：字符串键绑定、同名回退

**已解析调度器谓词**：
以谓词变量作为条件的 IF，其定义比较是同一调度器行中更早的当前指令。复制链顺序在该比较
处而非后续 IF 处衡量，因此比较后的状态复制绝不是重放证据。

**状态地址逃逸**：
将调度器状态的 `ADDRESS_OF` 或 `ADDRESS_OF_FIELD` 直接或经变量/holder 定义发布到内存或
未知内存效果操作。地址一旦被存储或保留，后续未知操作或非精确 store 即使没有显式指针
参数也可能重新取得并修改状态。

**具体调度器重放**：
保持原操作数顺序和位向量有符号性，通过实际调度器 CFG 路由一个恢复的
`(state_token, width)`。它证明该令牌的目标，不构建符号状态区间。
_避免使用_：符号范围恢复、假定比较臂

**状态变量**：
调度器读取以选择下一个原始基本块的值。

**调度器比较变量**：
调度器比较读取的行内根值，通常就是状态变量。若每个调度器入口都经过一个已证明的等宽整
变量 latch，它可为转移状态变量的刷新副本。

**共享状态 latch**：
由原始基本块写入的变量刷新调度器比较变量的、唯一调度器入口复制链。仅当至少两个独立的
调度器目标头区域拥有不同写入并馈入它时，才是调度器边界；OBB 局部状态选择合并不是共享
latch。

**状态令牌**：
与状态变量比较的不透明调度器值。位宽属于其身份；不得假定所有令牌均为 32 位。

**原始基本块**：
平坦化将其经调度器重定向前，来自原控制流的基本块。
_避免使用_：短代码注释以外的 OBB

**去平坦化**：
恢复调度器控制后继后，将原始基本块重新直接相连。

**去平坦化语义验收**：
以恢复后高层控制流仍表达原始程序谓词、且不再以不透明调度器状态令牌决定原始分支为准。
它不以 switch 展示形状、MLIL 形状或工作流阶段回执作为完成证据。
_避免使用_：显示为 switch、receipt 已关闭

**共享尾谓词重放**：
条件两臂保留共享语义尾部时，在其最终调度器出口重建原始条件的方式。仅当该条件是当前、
直接变量/常量比较，且变量在两臂和共享尾部没有可观察修改或逃逸时可用；否则保留由状态
令牌路由的出口。
_避免使用_：默认复制条件、按 token 强制恢复语义

**过时状态写入**：
因其所属转移被直接重定向到已恢复后继而证明不再必要的调度器状态赋值或存储。去平坦化
计划按精确当前 MLIL 指令索引识别它；其他位置匹配的令牌值或变量不是清理证据。
_避免使用_：状态令牌扫描、整函数清理匹配

**原子去平坦化替换**：
一个包含每个已选调度器出口或条件改写，以及每个精确过时状态写入 NOP 的 MLIL
copy-transform。任一已选改写无法应用时，不安装任何替换。

**无条件转移**：
由一个具体状态令牌选择的已恢复原始块后继；仅当区域中多个状态写入均解析为同一令牌时，
才允许这些写入。

**条件转移**：
由程序控制流在多个状态令牌中选择的已恢复原始块后继集合，例如分支/状态选择菱形。当前
已建模样本中，它携带两个分支结果，各有状态令牌和目标原始基本块。两分支目标均解析、每条
路径均建立其令牌且改写区域私有时，去平坦化会改写条件转移。不同调度器出口存在时它保留
分支执行；两臂汇合到同一私有语义尾部时它保留尾部，仅以已写状态令牌改写唯一调度器出口；
其他情况只有完整证明可绕过状态通道时才捷径化条件。

**工作流阶段**：
按函数恢复工作的固定阶段槽位，其结果决定后续恢复能否运行。

**触发重新分析的修改**：
可能再次安排函数分析、从而重新进入工作流的 Binary Ninja 函数状态编辑；只有 DispatchThis
核心可提交这类修改。

**阶段清理**：
间接分支或调用阶段达到稳定后，在当前 MLIL 上从所属 phase 的精确根反复规划并移除死亡
目标解码 IL。仅本轮未 NOP 且计划为空时标记回执完成；不得为确认清理主动安排重新分析。
计划重复或应用失败则保持回执开放并失败关闭。去平坦化状态写入 NOP 属于原子去平坦化
替换，不属阶段清理。分支条件翻译只为实际安装成功的站点提供精确目标解码根；只有 SSA
存活性决定其中哪些纯赋值已经死亡。

**当前分支清理固定点**：
同一份当前 MLIL overlay 中，已按精确 branch receipt 根局部清理且再次计算为空的状态。若
branch cleanup receipt 仅因本轮 translator 在已安装 overlay 中实际 NOP 且局部收敛而保持开放，
`cleanup_overlay_ready` 才允许 deflatten 使用这个当前固定点；它不是跨重新分析持久化的
receipt，也不能替代调用清理证明。
_避免使用_：旧 instruction index、已关闭 receipt 的同义词
