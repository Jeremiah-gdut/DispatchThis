# 未支持调用不阻塞后续阶段

调用阶段稳定只表示受支持的单目标改写和类型调整已经收敛、当前不会再提交调用修改；它不要求
函数中所有间接调用都成为 direct call。完整多目标或其他未支持调用保留当前 IL、活目标解码且
不创建 receipt，因此不阻止 global、分支翻译、关联 STORE、字符串或 deflatten 运行。间接调用
不同于未恢复间接分支，不会造成 CFG 缺边；真正依赖其目标或内存效果的下游计划必须在相关
站点或区域局部拒绝，而不是用函数级 gate 丢弃无关恢复进展。

因此，调用 provider 的 `CompleteBatch` 可以省略未匹配或当前不支持的调用，只要整轮候选扫描
可信且每个返回事实均为站点完整。只有扫描本身无法完成时使用 `Inconclusive` 并保持调用阶段
未收敛；省略不是证明失败的替代编码。
