# 以 CFG 入边见证关联 PHI 取值

完整值求解不能把相关 PHI 各自降为独立集合后计算笛卡尔积。核心以当前 CFG 的精确入边见证
作为同一汇合点的路径主键，按该主键联合 sibling PHI；跨汇合点的取值案例携带相容入边选择
的合取。重复的完整绑定只合并其路径来源，路径无关消费者才可投影为去重值集合；无法证明
operand 与入边的关系，或不同有效关联产生不同关系时，返回 `Inconclusive`，绝不猜测操作数
顺序、接受部分路径或回退笛卡尔积。这里的路径相容只表示入边选择能在 Binary Ninja 当前
CFG 上组成同一控制流经过关系；核心把该 CFG 路径范围视为既定输入，不判断分支谓词是否
可满足，也不识别虚假分支或求解不透明谓词。路径关联关系只在间接跳转目标计算期间存在；
目标事实和回执不保存 PHI case 或旧 CFG 入边。若 deinbr 由该关系证明了条件跳转，其最终
目标事实必须保留混淆前的语义条件以及 true/false 目标方向；后续分支条件翻译重新绑定当前
IL 见证并消费该结论，不重新计算目标或独立推断方向。
