# Scope analysis settings to enabled functions

DispatchThis will stop changing Binary Ninja analysis limits and builtin outlining settings when the plugin is imported. The earliest resolver workflow callback will apply the required values to its current `Function` with `SettingsResourceScope`, after workflow eligibility has established that DispatchThis is enabled. The plugin's fixed values take precedence over inherited or user values for that function because they are part of the expected analysis environment: unlimited function size, expression-value depth 99999, analysis time 1800000 ms (30 minutes), update count 1024, and builtin outlining disabled. The callback writes only mismatched values, verifies all five through the Function resource, and skips recovery for that run if the required environment cannot be established. Function overrides remain after DispatchThis is disabled; the plugin will not maintain prior-value state or implement an unreliable partial reset path. This keeps large obfuscated functions analyzable without changing analysis behavior for unrelated functions or BinaryViews.

Binary Ninja 5.3 or newer is the compatibility baseline for this ownership model.
