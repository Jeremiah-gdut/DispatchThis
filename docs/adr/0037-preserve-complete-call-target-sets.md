# 调用事实保留完整目标集合

`CallTargetFact` 以非空、去重的 `targets` 元组表达一个间接调用的全部已证明 callee，避免公开
provider API 从单值扩展到多值时再发生破坏性变更。首版后端只对单元素集合执行常量 destination
改写、类型调整、回执和目标解码清理；多目标事实保持原间接调用及其活目标解码，但不阻止调用
阶段收敛。Binary Ninja 没有与站点级间接分支目标对等的多 callee 提交 API，而变量定义级 UIDF
可能影响其他消费者，因此在真实样本证明安全边界前不增加第二条多目标修改路径，也绝不隐式
选择第一个 callee。下游只在自身证明真实依赖该调用时局部拒绝。
